# The Agentic Skill Manifesto

## What a skill is — and what it isn't

A skill is not a prompt. A skill is not a template. A skill is not a chatbot persona.

A skill is a **composable unit of expert judgment** — a self-contained agent that knows *what* to do, *when* to do it, *how* to verify it did it right, and *who* to hand off to when the task crosses its boundary.

## The 7 properties of a well-forged skill

1. **Bounded expertise** — It knows exactly what it owns and what it doesn't. No scope creep, no "I can also help with..."

2. **Opinionated defaults** — It ships with strong defaults derived from real operator experience, not generic best practices. Users override; they don't configure from scratch.

3. **Tool-native** — It doesn't describe actions; it takes them. MCP tools, file I/O, browser automation, API calls — a skill reaches through the interface into the system.

4. **Context-aware** — It reads the room. Project state, CRM data, previous decisions, calendar, active deals — it pulls what it needs before asking the user.

5. **Composable** — It calls other skills and gets called by them. The graph of skills forms an operating system, not a menu of options.

6. **Verifiable** — Every output has a definition of done. A skill knows when it succeeded and can prove it.

7. **Evolvable** — It learns from corrections. User overrides become future defaults. The skill gets sharper with use.

## The operating principle

Skills don't wait to be asked. They observe, infer, prepare, and propose. The founder's job is to decide — not to remember, retrieve, draft, format, check, or chase.

## The test

If removing a skill from the system creates a gap that a human would need to fill with 2+ hours of weekly work — it's a real skill. If it just reformats text — it's a prompt.
