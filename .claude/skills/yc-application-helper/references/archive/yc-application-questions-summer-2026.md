## Critical observations from this question set

### Versus historical YC applications (analytical commentary, not partner-sourced — to verify in Phase 2):

1. **Per-founder profile is now separate and gating** — partners see each founder via their independent profile completion, not consolidated in main form. The "tell us something impressive each founder built" lives here (per howtoapply page citation of PG calling it "the most important question").

2. **NEW Summer-2026-only question — coding agent session attachment.** Strong signal: YC values builders using modern AI tooling (Claude Code, Cursor). This question is application-defining for technical founders.

3. **Revenue is monthly-itemized over 6 months** — partner can see trajectory, not just total. Revenue stagnation across months is highly visible.

4. **Demo upload is REQUIRED, not optional** — "Required: 1-3 minutes / 100 MB". The pre-2026 framing of "submit a demo if you have one" has hardened. No demo = significant disadvantage.

5. **"Successfully hacked" wildcard question NOT visible in main form** — either moved to per-founder profile, or removed. Verify in Phase 2 by checking per-founder profile contents.

6. **"How will you make money?" framed loosely** — partners explicitly accept "best estimate," reducing weight on revenue model precision at idea stage.

7. **Idea-5 (other ideas considered) is partner-flagged as high-leverage** — quote: "Often when we fund people it's because of something they list here and not in the main application." Strongest verbatim partner signal in the entire form.

8. **Equity disclosure is precise** — investor name + amount + valuation + security type + date for each investment. Vague answers fail this gate.

9. **"Are you currently fundraising?" and details** — partners assess parallel-fundraise risk (you may close before YC accepts).

10. **Founder video AND product demo are distinct uploads** — must produce two separate videos. Confirm format/duration of founder video in Phase 2.

## Application atoms (preliminary — Phase 2 will multi-pass these)

- **APP-Q-001 (50-char description):** Most-test-density question. 50 chars ≈ 8-10 words. Partners use this to decide whether to keep reading. Anti-pattern: marketing-speak, jargon, "the X of Y" analogies.
- **APP-Q-002 (coding agent session — NEW):** Summer 2026 unique. Prepare a clean, demonstrable session export. AI-native builders should treat this as MANDATORY despite optional label — strong signal.
- **APP-Q-003 (monthly revenue trajectory):** Stagnant or declining months are visible. Pre-revenue companies fill zeros; growing months tell their own story.
- **APP-Q-004 (other ideas):** Highest leverage per partner-stated quote. Treat with same rigor as main idea answer. Pivot-readiness signal.
- **APP-Q-005 (location decision):** Partners weight SF-relocation commitment. Vague or hedged answers signal half-commitment.
- **APP-Q-006 (per-founder profile gating):** Submission blocked until each founder completes profile. Coordination overhead — start each founder's profile early.
