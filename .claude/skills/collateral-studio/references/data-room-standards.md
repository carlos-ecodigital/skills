# Data Room & Document Standards

Formatting, numbering, and classification standards for formal Digital Energy deliverables. Apply these standards to investor materials, partnership proposals, RFP responses, and any document that enters a data room or formal review process.

---

## 1. Document Classification

### Classification Levels

| Level | Label | Who Can See | Use Case |
|---|---|---|---|
| **PUBLIC** | No marking required | Anyone | Website copy, press releases, published articles |
| **EXTERNAL** | "External — [Recipient]" | Named recipients + DE team | Partnership proposals, pitch decks, one-pagers |
| **CONFIDENTIAL** | "Confidential" | DE team + counterparty under NDA | Financial models, term sheets, technical specifications |
| **INTERNAL** | "Internal Only" | DE team only | Strategy documents, board materials, war room memos |

### Classification Rules
- Every formal document must carry a classification label on the first page
- Classification appears in header or footer on every page
- When in doubt, classify UP (more restrictive), not down
- Reclassification requires CEO or legal approval
- Email attachments inherit the classification of the document, not the email

## 2. Document Numbering

### File Naming Convention

```
[DE]-[Category]-[Topic]-[Version]-[Date].[ext]
```

**Examples:**
- `DE-PROP-Lodewijk-Partnership-v3-20260115.pdf`
- `DE-DECK-Grower-Pitch-v2-20260201.pptx`
- `DE-1PGR-Investor-Overview-v1-20260210.pdf`

### Category Codes

| Code | Category | Description |
|---|---|---|
| PROP | Proposal | Partnership proposals, investment proposals |
| DECK | Presentation | Pitch decks, conference presentations |
| 1PGR | One-Pager | Single-page summaries and teasers |
| WPPR | Whitepaper | Technical whitepapers, research papers |
| CASE | Case Study | Project case studies, proof of concept summaries |
| RESP | Response | RFP/RFI responses |
| FACT | Fact Sheet | Project-specific or product-specific fact sheets |
| TEMP | Template | Reusable templates and frameworks |
| BRCH | Brochure | Marketing brochures and capability statements |

### Version Control
- Major versions: v1, v2, v3 (structural changes, new content sections)
- Minor versions: v1.1, v1.2 (corrections, data updates, formatting fixes)
- Draft versions: v0.1, v0.2 (internal review only; never send externally)
- Final versions: remove "DRAFT" watermark, lock editing, convert to PDF

## 3. Page Layout Standards

### Page Setup
- **Paper size:** A4 (210mm × 297mm) for European audiences; Letter (8.5" × 11") for US only
- **Margins:** 25mm minimum all sides; 30mm for bound documents
- **Font body:** Professional sans-serif (Helvetica, Arial, or brand font if defined)
- **Font size:** 10-11pt body text; 8-9pt footnotes; 14-16pt section headers; 20-24pt document title
- **Line spacing:** 1.15-1.3 for body text; single for tables and captions
- **Page numbers:** Bottom right, format "Page X of Y"

### Header/Footer
- **Header:** DE logo (left), document title (center), classification (right)
- **Footer:** Document reference number (left), date (center), page number (right)
- **First page:** Full title page with document metadata (no header/footer or separate design)

### Title Page Elements
1. DE logo
2. Document title
3. Subtitle/description (one line)
4. Recipient name and organization (for external documents)
5. Classification label
6. Version number and date
7. "Prepared by Digital Energy Group AG"
8. Confidentiality notice (for Confidential and above)

## 4. Content Formatting Standards

### Headings
- **H1:** Document sections (numbered: 1, 2, 3...)
- **H2:** Subsections (numbered: 1.1, 1.2, 1.3...)
- **H3:** Sub-subsections (numbered: 1.1.1, 1.1.2...) — use sparingly
- Maximum 3 heading levels in any document
- Heading text: sentence case ("Revenue stacking overview" not "Revenue Stacking Overview")

### Tables
- All tables must have a caption ("Table 1: Comparison of heat sources")
- Table headers: bold, left-aligned
- Numeric data: right-aligned within columns
- Currency: always specify (EUR, USD, CHF)
- Source footnote below table when data is from external sources

### Financial Data Presentation
- Always show three scenarios: Conservative / Base Case / Optimistic
- Include assumptions below the table
- Currency precision: thousands (EUR K) for project-level; millions (EUR M) for portfolio-level
- Percentages: one decimal place (12.5%, not 12.47%)
- IRR, DSCR, LLCR: two decimal places
- Include "Indicative — subject to project-specific analysis" disclaimer

### Charts and Diagrams
- Every chart must have a title, axis labels, and legend
- Source footnote below chart
- Use consistent color palette (align with DE brand colors if defined)
- Prefer simple chart types: bar, line, waterfall. Avoid 3D, pie charts, or decorative elements.

### Footnotes and Sources
- Numbered sequentially per page or per section
- Format: "1. Source: [Name], [Year]. [Brief context if needed]."
- For proof points from `de-brand-bible/references/proof-points.md`, use the listed source
- For internal analysis: "Source: DE internal analysis, [Year]."

## 5. Deliverable-Specific Standards

### Pitch Deck (DECK)
- 12-18 slides maximum (audience-dependent)
- No slide should have more than 40 words of body text (titles excluded)
- Every third slide minimum should contain a quantified data point
- Include slide numbers and document reference in footer
- Final slide: specific next step with contact details

### One-Pager (1PGR)
- Single A4 page, both sides allowed for detailed versions
- Top third: headline claim + one supporting visual
- Middle third: 3-4 key proof points with numbers
- Bottom third: "about DE" mini-boilerplate + contact + next step
- No more than 300 words total

### Proposal (PROP)
- Table of contents for documents >5 pages
- Executive summary on page 2 (max 1 page; non-technical; decision-ready)
- Section numbering throughout
- Appendices for detailed technical data, CVs, legal structure diagrams
- Sign-off page at end (if appropriate)

### Whitepaper (WPPR)
- Abstract/executive summary (200 words max)
- Numbered references section at end
- Charts and tables with full source attribution
- 6-12 pages (excluding references and appendices)
- Can include QR code linking to related resources

### RFP/RFI Response (RESP)
- Mirror the RFP's section structure exactly
- Cross-reference each requirement with DE's capability
- Compliance matrix: Requirement | DE Response | Evidence/Reference
- Include "additional value" section for capabilities beyond RFP requirements
- Executive summary that addresses the evaluator's top 3 likely concerns

## 6. Quality Checklist

Before any formal deliverable leaves DE:

### Content
- [ ] Every number has a source (proof-points library or project-financing skill)
- [ ] No banned phrases (check `de-brand-bible/references/banned-phrases.md`)
- [ ] Terminology is consistent (DEC, ProjectBV, recht van opstal — not mixed with informal alternatives)
- [ ] Financial projections show three scenarios
- [ ] Next step is concrete and time-bound
- [ ] Correct buyer persona addressed (language, concerns, decision criteria match)

### Formatting
- [ ] Classification label on every page
- [ ] Page numbers present
- [ ] File named per convention
- [ ] Version number and date on title page
- [ ] Headers and footers consistent
- [ ] Tables have captions and sources

### Review
- [ ] Spell-check in correct language (NL or EN)
- [ ] No confidential information in external documents
- [ ] Partner names used only with approval
- [ ] Legal disclaimer present where required
- [ ] PDF version created for distribution (editable source retained internally)

## 7. Confidentiality Notice (Standard Text)

### English
```
CONFIDENTIAL

This document is prepared by Digital Energy Group AG for the sole use of
[Recipient Name/Organization]. It contains proprietary information and may
not be reproduced, distributed, or disclosed to third parties without the
prior written consent of Digital Energy Group AG.
```

### Dutch
```
VERTROUWELIJK

Dit document is opgesteld door Digital Energy Group AG uitsluitend voor
gebruik door [Naam Ontvanger/Organisatie]. Het bevat vertrouwelijke
informatie en mag niet worden gereproduceerd, verspreid of openbaar gemaakt
aan derden zonder voorafgaande schriftelijke toestemming van Digital Energy
Group AG.
```
