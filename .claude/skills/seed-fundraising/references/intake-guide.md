# Intake Guide -- Seed Round Fundraising

## 1. Purpose

This guide defines the comprehensive intake process for seed-round fundraising. The intake serves three functions:

1. **Information gathering**: Collect every data point needed to produce all 9 deliverables
2. **Document ingestion**: Review existing materials before asking redundant questions
3. **Validation**: Cross-check answers for consistency, flag red flags, identify gaps

The intake is designed for infrastructure/energy/AI ventures. Questions use asset-backed metrics (MW, sites, grid connections, CAPEX/MW) rather than SaaS metrics (MRR, DAU, churn) unless the company is software-only.

## 2. Intake Principles

| Principle | Implementation |
|---|---|
| Document-first | Phase 0 ingests existing materials before interrogation |
| Layered | Core questions always asked; conditional questions triggered by answers |
| Grouped | Present 3-5 questions per interaction round (~15-17 rounds total) |
| Validated | Post-intake consistency check against benchmarks and cross-references |
| Infrastructure-native | Vocabulary adapted for asset-backed infrastructure companies |
| Deliverable-mapped | Every question maps to at least one deliverable output |

## 3. Phase 0: Document Ingestion & Triage

**Purpose**: Understand what already exists before asking questions. Prevents redundant interrogation and surfaces investor feedback early.

**Present as first interaction round.**

### Questions (0.1-0.8)

| Q# | Question | Format | Feeds |
|---|---|---|---|
| 0.1 | Do you have an existing pitch deck? If yes, share it for review. | File upload / paste | All deliverables |
| 0.2 | Do you have a financial model (Excel/Sheets)? If yes, share it or describe its structure. | File upload / describe | Financial projections, IM, data room |
| 0.3 | Do you have a cap table (spreadsheet or summary)? | File upload / describe | Cap table, IM Section 12 |
| 0.4 | Do you have a current executive summary, one-pager, or company overview? | File upload / paste | Exec summary, IM Section 1 |
| 0.5 | Have you received investor feedback? What did they say (positive and negative)? | Free text | Narrative strategy, risk framing |
| 0.6 | What is your highest priority right now? | Choice: narrative / financials / documentation / outreach / all | Deliverable sequencing |
| 0.7 | Which of these documents already exist? (Select all that apply) | Checklist | Data room readiness |
| 0.8 | Any specific investor questions or DD requests you need to address urgently? | Free text | Prioritization |

### Document Existence Checklist (Q0.7)

Present as a structured checklist. Mark each as: exists / in progress / doesn't exist / N/A.

**Corporate:**
- [ ] Certificate of incorporation / Handelsregister extract
- [ ] Articles of association (Statuten)
- [ ] Shareholders agreement (SHA)
- [ ] Board resolutions
- [ ] Organizational chart
- [ ] UBO register extract

**Financial:**
- [ ] Financial model (Excel/Sheets)
- [ ] Historical financial statements
- [ ] Bank statements (last 6 months)
- [ ] Cap table spreadsheet
- [ ] Existing SAFEs / convertible notes / loan agreements

**Legal:**
- [ ] Customer contracts or LOIs
- [ ] Supplier / EPC contracts or heads of terms
- [ ] Land agreements (recht van opstal, erfpacht, lease)
- [ ] Grid connection agreements or correspondence
- [ ] O&M contracts
- [ ] Patent filings or IP documentation
- [ ] Trademark registrations
- [ ] Permits (omgevingsvergunning, etc.)
- [ ] SDE++ beschikking
- [ ] Environmental assessments
- [ ] Privacy policy / GDPR documentation

**Technical:**
- [ ] Technical architecture document
- [ ] Site plans / layouts
- [ ] Grid connection technical specifications
- [ ] Equipment specifications (BESS, cooling, power)
- [ ] Independent engineer reports
- [ ] PUE analysis / energy efficiency data

**Team:**
- [ ] Founder CVs / biographies
- [ ] Key employee profiles
- [ ] Advisory board profiles
- [ ] Employment agreements (founders)
- [ ] ESOP / option plan documentation

**Market:**
- [ ] Market research reports (third-party)
- [ ] Competitive analysis document
- [ ] Customer references / testimonials
- [ ] Pipeline summary

**Fundraise:**
- [ ] NDA template
- [ ] SAFE or convertible note template
- [ ] Term sheet (received or drafted)

### Phase 0 Processing

After receiving Phase 0 answers:
1. **If existing deck provided**: Review and note what information is already captured vs. gaps
2. **If existing model provided**: Note structure, assumptions, revenue streams modeled
3. **If existing cap table provided**: Note shareholders, %, instruments, ESOP status
4. **If investor feedback provided**: Note objections/concerns to address in narrative
5. **Generate a gap list**: What the intake still needs to collect given existing materials
6. **Skip questions in Phases 1-6 that are already answered** by provided documents

## 4. Phase 1: Company Foundation

**Purpose**: Establish the corporate identity, entity structure, team, and governance. Feeds pitch deck Slides 1, 9; IM Sections 1-2, 9; exec summary header and team section; data room folders 02, 07.

**Present in 2 interaction rounds (1.1-1.4, then 1.5-1.8).**

### Questions (1.1-1.8)

| Q# | Question | Why It Matters | Feeds |
|---|---|---|---|
| 1.1 | Company name, legal entity type(s), jurisdiction(s), registration number(s) | Investors verify legal existence; entity type affects instrument choice | Deck Slide 1, IM Section 2, data room 02, jurisdiction-guide |
| 1.2 | Full entity structure: holding entity, operating entities (BVs/subsidiaries), JVs, intercompany arrangements | CH+NL dual-entity assumed by cap-table-guide; investors need structure clarity | Cap table, IM Section 2, jurisdiction-guide, data room 02 |
| 1.3 | One-sentence description (10-15 words) + one-paragraph description (50-75 words) | Different contexts need different lengths; both appear in materials | Deck Slide 1, exec summary header, elevator pitch, outreach emails |
| 1.4 | Founding date; incorporation date (if different); key milestone dates (first hire, first customer, first site, etc.) | Timeline shows velocity; Lochmiller mode needs specific dates | Deck Slide 6, IM Section 7, elevator pitch |
| 1.5 | For EACH founder: name, title/role, relevant experience (companies, roles, years), sector expertise, prior exits or notable achievements, % time committed to this company | Team is #1 factor at seed; pitch-deck Slide 9 needs 2-3 bullets per founder | Deck Slide 9, IM Section 9, exec summary team, outreach |
| 1.6 | Advisory board: names, domains of expertise, involvement level, any equity/compensation? | Advisors signal credibility and network; investors ask about governance | Deck Slide 9, IM Section 9, data room 07 |
| 1.7 | Current team: total headcount, breakdown by function (engineering, ops, biz dev, admin); next 3-5 key hires planned (role, timeline, budget per hire) | Team plan shows growth trajectory; use of proceeds needs hiring costs | Deck Slide 9, IM Section 9, financial projections (OPEX), use of proceeds |
| 1.8 | Board composition: who has seats? Any observer rights? How are major decisions made (simple majority, supermajority, veto rights)? | Governance cleanliness matters; investors need to know board dynamics before joining | IM Section 2, cap table, data room 02, SHA |

### Conditional Questions (Phase 1)

| Condition | Additional Question |
|---|---|
| If multi-entity structure (1.2) | Intercompany agreements: management fees, IP licensing, cost-sharing? Transfer pricing documentation? |
| If Swiss AG (1.1) | Authorized capital status? Conditional capital reserved for ESOP? Capital band mechanism used? |
| If Dutch BV(s) (1.1) | Notaris used for formation? Share classes (letteraandelen)? Distribution test compliance? |
| If no SHA exists (from 0.7 checklist) | Flag as gap: SHA should exist before fundraising; recommend drafting |

## 5. Phase 2: Problem, Solution & Moat

**Purpose**: Crystallize the narrative core. Feeds pitch deck Slides 2-3, 7; IM Sections 3-4; exec summary problem/solution; elevator pitch; all archetype voice assignments.

**Present in 2 interaction rounds (2.1-2.4, then 2.5-2.8).**

### Questions (2.1-2.8)

| Q# | Question | Why It Matters | Feeds |
|---|---|---|---|
| 2.1 | What is the customer's problem in their own words? Who feels this pain most acutely? Quantify the cost of the problem (EUR, time, MW lost, etc.) | Musk mode needs existential urgency with numbers; problem slide must resonate | Deck Slide 2, IM Section 3, exec summary, elevator pitch hook |
| 2.2 | What is your solution? Describe how it works technically in 3-4 sentences. What does the customer experience? | Must be concrete, not abstract; Lochmiller mode needs operational detail | Deck Slide 3, IM Section 4, exec summary solution |
| 2.3 | What are your top 3 differentiators vs. the status quo or closest alternative? (Be specific: "We do X while they do Y because Z") | Positioning matrix needs axes; competitive slide needs honest contrast | Deck Slide 3, Slide 8, IM Section 4, positioning |
| 2.4 | What is your moat / defensibility? Why can't a well-funded competitor replicate this in 12-18 months? | Moat analysis in sector-thesis.md has 6 types; need to map founder's answer | Deck Slide 7, IM Section 4, exec summary |
| 2.5 | IP status: any patents filed or granted? Trade secrets? Proprietary processes or algorithms? Trademarks registered? | Pitch deck Slide 7 and IM Section 4 expect IP detail; data room 04_Legal/IP | Deck Slide 7, IM Section 4, data room 04 |
| 2.6 | Technology readiness: is the core technology proven at commercial scale, piloted, prototyped, or conceptual? What validation exists? | Calibrates risk framing; Liebreich mode needs evidence-based claims | IM Section 4, deck Slide 7, risk assessment |
| 2.7 | What is the founder origin story? Why did you start this company? Why you, why now? | Elevator pitch and deck opening need authentic narrative; Musk mode personal conviction | Elevator pitch, deck Slide 2, IM Section 2, outreach |
| 2.8 | What is the 5-10 year vision? If everything goes right, what does this company become? | Deck Slide 12 (vision); investment thesis framing; Musk + Wood modes | Deck Slide 12, IM Section 4 (roadmap), elevator pitch close |
| 2.9 | Product/platform roadmap: what's built and operational today, what's in development for the next 12-18 months, and what's the long-term platform vision? | IM Section 4 requires current/near-term/long-term roadmap; deck Slide 7 needs product trajectory | IM Section 4, deck Slide 7, exec summary |
| 2.10 | What are the top 3 risks to your business? For each: what category (market, technology, regulatory, execution, financial), how likely, how severe, and what's your mitigation? | IM Section 13 requires structured risk table; exec summary 2-page needs top 3 risks; Liebreich mode | IM Section 13, exec summary (2-page), investor Q&A prep, risk assessment |
| 2.11 | What is the strongest "hook" to open a pitch -- a surprising fact, provocative stat, or urgent data point that makes someone stop and listen? | Elevator pitch opens with hook; outreach email subject line needs this; Musk mode opening | Elevator pitch, outreach emails, deck Slide 2, exec summary lead |

## 7. Phase 4: Assets, Traction & Operations

**Purpose**: Map the physical asset pipeline, quantify traction, and capture operational reality. This is the most infrastructure-specific phase and has NO equivalent in SaaS fundraising intakes. Feeds pitch deck Slides 6-7; IM Sections 7, 10; financial projections (per-site inputs); data room folders 04-06.

**Present in 3 interaction rounds (4.1-4.4, then 4.5-4.8, then 4.9-4.13).**

### Questions (4.1-4.13)

| Q# | Question | Why It Matters | Feeds |
|---|---|---|---|
| 4.1 | How many sites are in your pipeline? For each site, what is the status? Use: identified / LOI signed / permitted / grid-connected / under construction / operational | Site pipeline IS the business; financial model phases revenue by site status | Financial projections, deck Slide 6, IM Section 7, data room 05 |
| 4.2 | For each active site: location (municipality/province), grid connection capacity (MW), grid operator (TenneT / Liander / Stedin / Enexis / other), connection status (existing / applied / granted / energized), estimated COD | Grid connection is the #1 moat claim; must be verified and specific | Financial projections, deck Slide 7, IM Section 4, data room 04-05 |
| 4.3 | Land status per site: type of agreement (ownership / recht van opstal / erfpacht / lease / option), counterparty, duration, cost (EUR/year or one-time), any conditions or restrictions | Land rights underpin every site; data room 04 needs agreements | Data room 04, financial projections (land OPEX), IM Section 7 |
| 4.4 | Revenue streams per site: which of the following apply? (BESS arbitrage, BESS ancillary/FCR/aFRR, AI colocation, heat supply, grid services/congestion management, other) | Revenue model is per-stream per-site; financial-projections.md formula requires this | Financial projections, deck Slide 5, IM Section 6 |
| 4.5 | Customer traction: for each signed contract or LOI, provide counterparty type, contract type, value (annual or total), term (years), key conditions | Traction hierarchy: signed contracts > LOIs > pipeline; investors weight contracted revenue | Deck Slide 6, IM Section 7, exec summary traction, data room 06 |
| 4.6 | Partnership traction: grower partners (who, hectares, heat demand), utility/DSO partners, technology partners, EPC/construction partners | Partnerships signal execution readiness; Lochmiller mode needs specifics | Deck Slide 6, IM Section 7, data room 06 |
| 4.7 | Revenue status: any current revenue? From which streams? Monthly or quarterly amount? Growth trajectory? | Revenue > everything else in traction hierarchy; even small revenue changes the narrative | Deck Slide 6, financial projections, exec summary, IM Section 7 |
| 4.8 | Key milestones ACHIEVED (with dates): permits granted, grid connections secured, first customer signed, first revenue, partnerships formalized, team hires, any other | Milestone timeline shows velocity; Lochmiller mode: "We did X in Y months vs. industry Z" | Deck Slide 6, IM Section 7, elevator pitch, outreach |
| 4.9 | Key milestones UPCOMING (with target dates): next 12-18 months, tied to use of proceeds where possible | Milestone plan shows what this round enables; Series A readiness criteria | Deck Slide 11, IM Section 11, exec summary, financial projections |
| 4.10 | Current monthly burn rate? Current cash balance? How many months of runway at current burn? | Cannot size round without this; financial-projections.md runway calculation | Financial projections, IM Section 10-11, cap table |
| 4.11 | Current team payroll as % of total burn? Top 3 non-payroll cost categories and monthly amounts? Key vendor/contractor relationships? | Cost structure feeds OPEX model; investors assess cost discipline | Financial projections, IM Section 6, deck Slide 5 |
| 4.12 | Any press coverage, awards, industry recognition, or notable customer/partner testimonials? | Social proof strengthens traction narrative; exec summary and outreach | Deck Slide 6, exec summary, outreach, data room 06 |
| 4.13 | What is the single strongest proof point you'd lead with to an investor? Why? | Forces founder to prioritize; this becomes the first line of every outreach email | Elevator pitch, exec summary lead, outreach, deck Slide 6 |

### Conditional Questions (Phase 4)

| Condition | Additional Question |
|---|---|
| If BESS operational or under construction | What arbitrage/ancillary revenue has been achieved? Actual vs. projected? Battery cycling data? Degradation observed? |
| If DC operational or under construction | Current PUE? Occupancy %? Customer pipeline for remaining capacity? |
| If heat supply active | Thermal MW delivered? Hours of operation? Customer satisfaction data? Price vs. fossil gas alternative? |
| If pre-revenue | What is the most advanced commercial discussion? Expected timeline to first revenue? |
| If multiple sites | What is the replicability model? Site 1 template -> Site N? What changes per site vs. what's standardized? |

## 8. Phase 5: Financials, Structure & Cap Table

**Purpose**: Capture the financial architecture of the raise and the company. Feeds pitch deck Slide 11; IM Sections 6, 10-12; financial projections (all tabs); cap table (pre/post); data room folder 03.

**Present in 3 interaction rounds (5.1-5.4, then 5.5-5.8, then 5.9-5.13).**

### Questions (5.1-5.13)

| Q# | Question | Why It Matters | Feeds |
|---|---|---|---|
| 5.1 | How much are you raising? (target amount, with acceptable range if flexible) | Defines the round; everything else sizes to this | All deliverables |
| 5.2 | Pre-money valuation expectation? Or say "help me think about this" and I'll benchmark against sector-thesis.md comparables (EUR 5-15M typical seed, +0-50% for asset-backed infrastructure) | Valuation determines dilution; must be defensible | Cap table, deck Slide 11, IM Section 12 |
| 5.3 | Instrument preference: SAFE, convertible note, priced equity round, or unsure? If unsure, I'll recommend based on round size and investor type. Any specific terms in mind (cap, discount, interest rate, maturity)? | Instrument choice affects legal complexity, investor type, and cap table mechanics | Cap table, IM Section 12, data room 03, 08 |
| 5.4 | Use of proceeds: for each major category, provide the allocation (amount or % of round). Categories to consider: team/hiring, BESS CAPEX, DC CAPEX, heat infrastructure, grid connection, working capital, professional services (legal/accounting/advisory), other | "Top 3-5 items" is too vague; financial-projections.md needs allocation with milestone mapping | Deck Slide 11, IM Section 11, financial projections, exec summary |
| 5.5 | Runway target: how many months should this round fund? What milestones must be achieved before Series A? When would you start Series A fundraising? | 18-24 months standard; cash flow model needs this; Series A trigger is critical | Financial projections, IM Section 11, deck Slide 11 |
| 5.6 | Revenue model detail per stream. For each revenue stream, provide: pricing ($/MW/yr, $/kW/mo, EUR/MWh, etc.), contract structure (fixed/variable, term, escalation), typical counterparty type, merchant vs. contracted split | Financial-projections.md formula requires per-stream revenue drivers; insufficient without this | Financial projections, IM Section 6, deck Slide 5 |
| 5.7 | CAPEX estimates per site. For each planned site, estimate: BESS CAPEX (EUR/MW), DC shell + core (EUR/MW), DC fit-out (EUR/MW incremental), grid connection upgrade (if needed), heat infrastructure, site development/land | Financial model phases CAPEX by site and quarter; without estimates, projections are fiction | Financial projections, IM Section 10, deck Slide 11 |
| 5.8 | Construction timeline per site: planned start, key milestones (EPC mobilization, equipment delivery, commissioning, COD), and estimated duration for each phase | Revenue start dates depend on construction timeline; delays are the #1 downside scenario | Financial projections, IM Section 7, scenario analysis |
| 5.9 | Current cap table -- DETAIL REQUIRED. For each shareholder: name (or category), share class, number of shares, % ownership. Include: founders (each individually), any angel investors, any institutional investors, any SAFE/note holders (amount invested, cap, discount, maturity). | Cap-table-guide needs each founder's %, instrument terms, ESOP status to model dilution | Cap table, IM Section 12, data room 03 |
| 5.10 | ESOP / option pool: does one exist? If yes: pool size (% of fully diluted), shares allocated vs. unallocated, vesting schedule, exercise price, any grants made. If no: is there a plan to create one? | Investors will insist on ESOP; if it doesn't exist, they'll require it pre-round (diluting founders) | Cap table, IM Section 12, financial projections |
| 5.11 | Share class structure: one class (ordinary) or multiple? Par value per share? Authorized share capital vs. issued? Any special rights attached to any class (voting, dividend, liquidation preference)? | Swiss AG and Dutch BV have different mechanics; cap-table-guide Section 5-6 needs this | Cap table, jurisdiction guide, data room 02 |
| 5.12 | Existing debt or obligations: any loans, credit facilities, guarantees, convertible instruments outstanding? If yes: counterparty, amount, terms, maturity, conversion provisions. Any personal guarantees by founders? | Debt impacts valuation and cap table; investors need clean structure or clear terms | Cap table, IM Section 10, data room 03, risk assessment |
| 5.13 | Exit expectations: what is the likely exit path (trade sale to utility/infra fund, platform IPO, secondary sale, merger, refinancing)? Expected timeline (3-5 years, 5-7 years, 7-10 years)? Target return profile (MOIC, IRR) for this round? | Every investor asks about exit; IM expects this; return profile affects investor type targeting | IM Section 12, deck Slide 12, investor targeting, outreach |

## 10. Post-Intake: Validation & Gap Analysis

After completing all phases, perform five validation steps before generating the Company Profile Summary or any deliverable.

### 10.1 Internal Consistency Check

Cross-reference answers across phases for contradictions:

| Check | Compare | Red Flag |
|---|---|---|
| Round size vs. use of proceeds | Q5.1 vs. Q5.4 | Allocations don't sum to round size |
| Round size vs. runway | Q5.1 vs. Q5.5 at burn rate Q4.10 | Round funds <15 months or >30 months |
| Revenue projections vs. site timeline | Q5.6 vs. Q5.8 | Revenue from sites not yet constructed |
| Cap table vs. instrument | Q5.9 vs. Q5.3 | Instrument doesn't match investor type in Q6.3 |
| Milestones vs. use of proceeds | Q4.9 vs. Q5.4 | Milestones require spend not in use of proceeds |
| Team plan vs. burn rate | Q1.7 vs. Q4.10 vs. Q5.4 | Planned hires exceed hiring budget |
| Traction claims vs. documents | Q4.5-4.6 vs. Q0.7 | Claims of signed contracts but no contracts in document checklist |
| Valuation vs. traction | Q5.2 vs. Q4.7 | High valuation with no revenue and no signed contracts |
| Grid connection claims vs. documentation | Q4.2 vs. Q0.7 | Claims of grid connection but no connection agreement exists |
| Site count vs. CAPEX | Q4.1 vs. Q5.7 | CAPEX per site exceeds seed round budget |

### 10.2 Benchmark Comparison

Compare key metrics against sector-thesis.md benchmarks:

| Metric | Company Value | Benchmark Range (sector-thesis.md) | Flag If |
|---|---|---|---|
| Pre-money valuation | From Q5.2 | EUR 5-15M (+0-50% asset-backed) | >EUR 20M with no revenue |
| BESS CAPEX per MW | From Q5.7 | EUR 330-700K/MW | Outside range by >20% |
| DC CAPEX per MW | From Q5.7 | EUR 8-12M/MW | Outside range by >20% |
| Heat revenue per MWh | From Q5.6 | EUR 15/MWh indicative | >2x or <0.5x benchmark |

## 12. Intake Completion Criteria

The intake is considered COMPLETE when:

- [ ] All 6 phases completed (Phase 0-6)
- [ ] All conditional questions triggered by answers have been asked
- [ ] Internal consistency check passed (Section 10.1) -- no unresolved contradictions
- [ ] Benchmark comparison completed (Section 10.2) -- outliers explained or corrected
- [ ] Red flags identified and discussed (Section 10.3)
- [ ] Document readiness scored (Section 10.4)
- [ ] Deliverable recommendation provided (Section 10.5)
- [ ] Company Profile Summary generated and approved by user

The intake is considered SUFFICIENT (for producing a specific deliverable) when:
- All primary questions for that deliverable (Section 11) have been answered
- No critical red flags remain unresolved for that deliverable's content
