# Pattern: Broadcast

## Purpose

One-to-many communication. A leader speaks; the team listens. The goal is alignment, energy, and shared context — not discussion, not status updates, not problem-solving.

Use this pattern when:
- A leader needs to set direction, frame priorities, or share strategic context
- The audience is too large for meaningful round-table discussion
- Written pre-reads have replaced verbal status updates and you need a live moment to connect them

Do NOT use this pattern when:
- You need the group to make decisions (use Decision)
- You need individual status updates (use Standup or async written updates)
- You need deep discussion on a topic (use Workshop)

## Format

1. **Leader opens** — wins, risks, one priority. Sets the emotional frame.
2. **Brief context** — connect this week to the bigger picture. What changed? What matters now?
3. **Targeted call-outs** — name specific people, specific actions, specific deadlines. Generic praise is noise.
4. **Q&A window** — short. Questions only. If a question turns into a discussion, park it.
5. **Leader closes** — restate the one priority. End with energy, not logistics.

The leader does 80%+ of the talking. That's the point. If everyone is talking equally, you're in the wrong meeting type.

## Duration

15-30 minutes. If it takes longer, you're doing status updates verbally — which means your pre-read system is broken.

## Frequency

Weekly or monthly. Weekly for operational alignment. Monthly for strategic resets.

## Key Rules

1. **No round-robin updates.** Written pre-reads replace verbal status. Anyone who starts "This week I..." is doing it wrong.
2. **No technical deep-dives.** If something needs >2 minutes of explanation, it belongs in a Workshop or Decision meeting.
3. **Pre-reads are mandatory input.** The leader reads them before the meeting and synthesizes. The team reads them before the meeting and comes with questions, not presentations.
4. **One priority, stated twice.** Open with it, close with it. If the team leaves remembering one thing, it should be the priority.
5. **Respect the clock.** Start on time. End early if you can. Never run over — it signals that your format is broken.

## Run-of-Show

| Segment | Time | Who | What |
|---------|------|-----|------|
| Opening | 3-5 min | Leader | Wins, risks, one priority. Set the frame. |
| Context | 3-5 min | Leader | This week in the bigger picture. What changed. |
| Call-outs | 5-10 min | Leader | Name people. Name actions. Name deadlines. |
| Q&A | 3-5 min | Team → Leader | Questions only. No debates. Park discussions. |
| Close | 2-3 min | Leader | Restate priority. End with energy. |

**Total: 16-28 minutes**

## Quality Check

After the meeting, ask:
1. Can every attendee state the one priority? If not, the message didn't land.
2. Did anyone give a verbal status update? If yes, fix the pre-read system.
3. Did the leader name specific people with specific actions? If not, it was too abstract.
4. Did it run under 30 minutes? If not, something leaked in that doesn't belong.
5. Did the energy go up or down? A broadcast should leave people more aligned and more motivated than when they walked in.
