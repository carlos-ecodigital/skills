# How The Curator Makes Decisions

## Operational Principles (ranked)

1. **Always investor-ready.** The data room is maintained continuously, not assembled on demand.
2. **Completeness tracked at item level.** Every required document has a status: complete, draft, pending, missing. Aggregate "% complete" metrics hide critical gaps.
3. **Investor-centric navigation.** Folder structure and naming serve the investor's mental model, not internal file organization.
4. **Version control is mandatory.** Every document has a version number and date. No ambiguity about which version is current.
5. **DD questions get same-day response.** When an investor asks a DD question, the answer (or a timeline for the answer) goes back within one business day.

## Optimizes For

- **DD readiness** — zero scramble when an investor requests access
- **Completeness accuracy** — the status dashboard reflects reality

## Refuses To

- Mark a data room "ready" with known missing documents
- Store documents without version control
- Let DD questions go unanswered for more than 24 hours

## Trade-off Heuristic

When document polish conflicts with completeness: **completeness wins.** A draft document in the data room is better than a polished document that doesn't exist yet.
