# Value Equation Framework — Adapted for Digital Energy B2B Infrastructure

Based on Alex Hormozi's $100M Offers value equation, adapted for B2B energy and AI infrastructure sales.

---

## The Equation

```
VALUE = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)
```

**Goal:** Maximize value by increasing the numerator (dream outcome, perceived likelihood) and decreasing the denominator (time delay, effort/sacrifice).

**Key insight for B2B:** The denominator matters more than the numerator in infrastructure sales. Sophisticated buyers already understand the dream outcome. What differentiates is: "How fast?" and "How easy?" If DE can make the process faster and require less effort than alternatives, the offer wins even if the dream outcome is comparable.

## Time Delay — Speed Advantages

| Buyer Ask | DE's Speed | Alternative's Speed | Advantage |
|---|---|---|---|
| Grower: "How fast to an LOI?" | 10 business days (portal) | 4-6 weeks (traditional) | 3-4x faster |
| Grower: "How fast to HoT?" | 6 weeks from LOI | 8-16 weeks (traditional) | 2x faster |
| BESS: "When operational?" | 6-12 months from FID | Similar | Parity, but starts before DC |
| Neocloud: "When can I deploy?" | Fonti: 4 months. Scale: 12-18 months | Self-build: 3-5 years | 3-10x faster |
| District heating: "When does heat flow?" | 18-30 months (DC development) | Geothermal: 3-5 years | 2x faster (and no drilling risk) |

**Always name the specific timeline.** "4 months from FID to operational" is more powerful than "fast deployment."

## Effort & Sacrifice — Friction Reduction

| Buyer Friction Point | How DE Minimizes It |
|---|---|
| Grower: "I have to fill out complex forms" | 30-second signup; 65%+ fields auto-filled by API enrichment; magic link auth |
| Grower: "I need to invest capital" | Zero CAPEX from grower. DE finances everything. |
| Grower: "I need to understand complex legal English" | Bilingual Dutch/English LOI and HoT. Plain language. |
| Neocloud: "I need to find grid capacity" | Grid is already secured. No 10-year wait. |
| Neocloud: "I need to build a facility" | Powered shell available — bring hardware, we provide infrastructure |
| Enterprise: "I need SOC 2 certification" | Built into delivery plan; achievable within build timeline |
| District heating: "Municipal board approval takes forever" | Pre-packaged board presentation materials with technical and financial documentation |
| Industrial: "Switching heat sources risks production" | Dual-source design: DEC primary, existing boiler retained as backup |

**The best friction reduction: eliminate a step entirely.** "Zero CAPEX" is more powerful than "low CAPEX." "Pre-filled forms" is better than "simple forms."

## Applying the Value Equation — Process

When designing any offer or campaign:

1. **Score current state** (1-10 for each lever: dream outcome, perceived likelihood, time delay, effort)
2. **Identify the weakest lever** — this is where improvement has the most impact
3. **Design interventions** for the weakest lever
4. **Re-score** after interventions
5. **Test** — pilot the improved offer against the current one

## Grand Slam Offer Design

A "Grand Slam Offer" (Hormozi) combines:
1. **The core deliverable** (heat supply, compute capacity, BESS returns)
2. **Bonuses that reduce time delay** (fast-track permitting, pre-approved financing structure)
3. **Bonuses that reduce effort** (portal automation, pre-filled contracts, dedicated project manager)
4. **Risk reversal** (grower: "If heat delivery drops below agreed minimum, price adjusts automatically")
5. **Urgency/scarcity** (real, not manufactured): "We have 3 grid connections available. Pipeline has 12 interested parties."

**For DE, scarcity is real.** Grid connections are genuinely scarce. Pipeline capacity is genuinely limited. This is not manufactured urgency — it's market reality. Use it honestly.
